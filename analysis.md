# Анализ рисков текущего подхода

При выполнении работы были выявлены следующие проблемы:

1. **Отсутствие атомарности:** В первом коммите смешаны файлы кода (`app.py`) и файлы зависимостей (`requirements.txt`). Если в будущем нам понадобится отменить изменения только в коде, нам придется откатывать и настройки окружения, что неудобно.
2. **Проблемы код-ревью:** Когда в одном коммите много разноплановых изменений, другому разработчику (или преподавателю) сложно понять логику правок.
3. **Воспроизводимость:** В файле `requirements.txt` указана библиотека `requests` без версии. Это риск: если выйдет версия `requests` 3.0, которая несовместима с моим кодом, проект перестанет работать на других компьютерах.
